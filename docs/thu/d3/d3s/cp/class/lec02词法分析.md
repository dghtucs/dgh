

词法分析的核心任务是从左到右读入源程序的字符流，识别出一个个的单词。所识别的每一个单词，是下一个有意义的词法元素，如标识符或整数。在识别出下一单词，同时也验证了其词法正确性之后，词法分析程序就会产生一个单词记录，传递给后续阶段使用

词法分析程序所产生的单词记录通常由两部分信息组成：一个是单词符号（token），对应某个特定意义的词法单元，如标识符、整常数等；另一部分是单词的属性值（attribute）




语法分析

语法分析的任务是根据语言的语法规则，识别出由词法分析器传递过来的单词流所组成的语句结构。语法分析器通常采用自顶向下或自底向上的方法，构造出源程序的语法树或抽象语法树

语法分析器需要处理的主要问题包括：
1. 识别语法结构：根据语言的语法规则，识别出各种语句、表达式等结构
2. 处理语法错误：当遇到不符合语法规则的单词流时，语法分析器需要能够检测并报告语法错误
3. 构造语法树：在识别出语法结构后，构造出相应的语法树或抽象语法树，供后续的语义分析和代码生成使用
4. 符号表管理：在语法分析过程中，维护符号表，记录变量、函数等的定义和作用域信息
5. 支持上下文无关文法：大多数编程语言的语法可以用上下文无关文法描述，语法分析器需要能够处理这类文法
6. 提高效率：语法分析器需要在保证正确性的前提下，提高分析速度，以适应大型程序的编译需求
7. 错误恢复机制：在遇到语法错误时，语法分析器应具备一定的错误恢复能力，尽量继续分析后续代码，而不是立即终止编译过程
8. 与词法分析器的接口：语法分析器需要与词法分析器紧密配合，正确获取单词流，并处理词法分析器可能产生的错误

通过有效的语法分析，编译器能够正确理解源程序的结构，为后续的语义分析和代码生成奠定基础
