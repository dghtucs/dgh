停等协议中，发送方发送一个报文段后会等待接收方的确认（ACK），若未收到确认则超时重传。若接收方收到重复报文段后仅丢弃而不做任何操作，会导致 死锁 ：
例如：
1. 发送方发送报文段`M1`，接收方正确接收并回复确认`ACK1`，但`ACK1`在传输中丢失；
2. 发送方超时后重传`M1`，接收方收到重复的`M1`，丢弃且不回复任何信息；

### 第10题

停等协议的效率$\eta = \frac{\text{帧传输时间}}{\text{帧传输时间} + 2 \times \text{单向传输时延}} \geq 50\%$。


- 信道速率$R = 4\text{kb/s} = 4000\text{b/s}$；
- 单向传输时延$T = 20\text{ms} = 0.02\text{s}$，往返时延$2T = 0.04\text{s}$；
- 设帧大小为$L$（单位：bit），帧传输时间为$\frac{L}{R}$。



代入效率公式并解不等式：
$$
\frac{\frac{L}{4000}}{\frac{L}{4000} + 0.04} \geq 0.5
$$
化简得：
$$
\frac{L}{4000} \geq 0.5 \times \left(\frac{L}{4000} + 0.04\right) \implies \frac{L}{8000} \geq 0.02 \implies L \geq 160\text{bit}
$$

因此，帧的大小需 不小于160bit 。

### 14题


-  传播时延 ：信号传播时间 = 链路长度 × 传播速度
  $t_{\text{传播}} = 2000\ \text{km} × 6\ \mu\text{s/km} = 12000\ \mu\text{s} = 12\ \text{ms}$

-  传输时延 ：帧的传输时间 = 帧长（比特）÷ 链路速率
  帧长 = $64\ \text{字节} = 64×8 = 512\ \text{bit}$，链路速率 = $1024\ \text{kbit/s} = 1024×1000\ \text{bit/s}$
  $t_{\text{传输}} = \frac{512\ \text{bit}}{1024×1000\ \text{bit/s}} = 0.0005\ \text{s} = 0.5\ \text{ms}$

-  往返时延（RTT） ：$RTT = 2×t_{\text{传播}} = 2×12\ \text{ms} = 24\ \text{ms}$



选择重传协议的窗口需覆盖 往返时延内可发送的帧数 ：
- 往返时延内链路可传输的总比特数（时延带宽积）：
  $\text{总比特数} = \text{速率} × RTT = 1024×1000\ \text{bit/s} × 0.024\ \text{s} = 24576\ \text{bit}$
- 最大窗口大小（可连续发送的帧数）：
  $W = \frac{\text{总比特数}}{\text{每帧比特数}} = \frac{24576}{512} = 48$



选择重传协议的窗口限制： 发送/接收窗口大小不超过 $2^{n-1}$ （$n$ 为序号位数），需满足 $2^{n-1} ≥ W$。

代入 $W=48$，计算最小的 $n$：
- $2^5=32 < 48$，$2^6=64 ≥ 48$，因此 $n-1=6$，即 $n=7$。



序号应该有 $\boldsymbol{7}$ 位。

### 第7题
CSMA/CD的最小帧长要求 帧传输时间≥往返传播时延，公式为：
$$\text{最小帧长} = \text{数据传输速率} × \text{往返传播时延}$$

- 单程传播时延：$\frac{10×1000\ \text{m}}{200\ \text{m/μs}} = 50\ \text{μs}$，往返传播时延：$2×50=100\ \text{μs}$；
- 数据传输速率：$10\ \text{Mb/s}=10×10^6\ \text{bit/s}$；
- 最小帧长：$10×10^6\ \text{bit/s} × 100×10^{-6}\ \text{s} = 1000\ \text{bit}$。


### 第8题
CSMA/CD中，最小帧长与站点距离的关系为：$\text{最小帧长} = R × 2×\frac{d}{v}$（$R$为速率，$d$为站点距离，$v$为传播速度），变形得：$d = \frac{\text{最小帧长} × v}{2R}$。

当最小帧长减少$800\ \text{bit}$时，距离变化量为：
$$\Delta d = \frac{(-800) × v}{2R}$$

代入$R=1\ \text{Gb/s}=1×10^9\ \text{bit/s}$、$v=200\ \text{m/μs}=2×10^8\ \text{m/s}$：
$$\Delta d = \frac{-800 × 2×10^8}{2×1×10^9} = -80\ \text{m}$$

即最远站点之间的距离应 减少80m 。

  - 端口2→网段1→根网桥A，路径开销=1；
  - 端口3→网段2→网桥C→网段1→根网桥A，路径开销=1+1+1=3；
  因此 网桥B的根端口是端口2 （开销更小）。

-  网桥C ：
  - 端口4→网段1→根网桥A，路径开销=1；
  - 端口5→网段2→网桥B→网段1→根网桥A，路径开销=1+1+1=3；
  因此 网桥C的根端口是端口4 （开销更小）。


### 步骤3：选择每个网段的指定端口
指定端口是 网段到根网桥路径开销最小 的端口
-  网段1 ：
  包含端口（A的端口1、B的端口2、C的端口4）。根网桥的端口（A的端口1）默认是指定端口（根到自身开销为0，小于B、C端口到根的开销1），因此 网段1的指定端口是A的端口1 。

-  网段2 ：
  包含端口（B的端口3、C的端口5）。
  - 端口3→网桥B→根网桥A，路径开销=1+1=2；
  - 端口5→网桥C→根网桥A，路径开销=1+1=2；
  开销相同，比较Bridge ID：网桥B的优先级（32769）小于网桥C（32779），因此 网段2的指定端口是B的端口3 。


### 步骤4：阻塞非指定端口
未被选为根端口/指定端口的端口需要阻塞，以消除环路：
- 网桥C的端口5未被选为根端口/指定端口，因此 阻塞网桥C的端口5 。


### 最终生成树结构
- 根网桥：网桥A；
- 根端口：网桥B的端口2、网桥C的端口4；
- 指定端口：网桥A的端口1、网桥B的端口3；
- 阻塞端口：网桥C的端口5。
